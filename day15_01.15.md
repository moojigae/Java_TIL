## 입출력(IO)
- 컴퓨터 내부 또는 외부 장치와 프로그램 간의 데이터를 주고 받는 것

### 스트림
- 프로그램과 컴퓨터 간의 연결하는 길
- 모든 스트림은 단방향이며 각각의 장치마다 연결할 수 있는 스트림 존재 - 하나의 스트림으로 입출력을 동시에 수행할 수 없으므로 동시에 수행하려면 2개의 스트림 필요
- 스트림 구분은 `바이트 기반 스트림(1byte)`과 `문자 기반 스트림(2byte)`으로 나눠져 있고 그 안에서 입력 스트림인지 출력 스트림인지 나뉘어짐
- 바이트 기반은 한계가 있음
- 000스트림 = 바이트 기반/입력(Input)/출력(Output)
- 000 = 문자 기반/입력(Reader)/출력(Writer)

#### 스트림 종류
- 기반 스트림 : 실제 스트림에 데이터가 왔다 갔다 하는 것
- 보조 스트림 : 기반 스트림의 성능 향상이나 기능 추가하거나 하는 보조를 해주는 스트림/ 데이터가 왔다 갔다 하지는 않음
- 그 클래스의 생성자 매개변수를 통해 기반 스트림인지 보조 스트림인지 알 수 있음 
	- 보조 스트림 : InputStream, OutpurStream, Reader, Writer가 매개변수로 들어감
- 스트림을 열었으면 꼭 닫아주기
	- 안닫아도 JVM이 자동으로 닫아주기 때문에 에러가 발생하지는 않으나 스트림이 많은 경우 그 때 그 때 닫지 않으면 그 안에서 데이터가 꼬일 수 있음 
	- 에러가 나더라도 꼭 닫아줘야 하기 때문에 finally 안에서 닫아야 함

> InputStream
- 바이트 기반 입력 스트림의 최상위 클래스로 추상클래스 임 
- 메소드 read / 매개변수 byte

> OutputStream
- 바이트 기반 출력 스트림의 최상위 클래스로 추상클래스 임 
- 메소드 write / 매개변수 byte

> Reader 
- 문자 기반 입력 스트림의 최상위 클래스로 추상클래스 임 
- 메소드 read / 매개변수 char

> Writer
- 문자 기반 출력 스트림의 최상위 클래스로 추상클래스 임
- 메소드 write / 매개변수 char or String
