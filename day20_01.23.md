### Stack : 후입선출(LIFO : Last In First Out) 구조로 JVM Stack 메모리가 Stack 구조로 되어 있음
### Queue : 선입선출(FIFO : First In First Out) 구조로 작업 큐나 메시지 큐가 Queue구조로 되어 있음
### Deque : 큐와 스택의 성질을 모두 가지고 있는 구조로 검색과 같은 반복적인 문제에 특화 되어 있음.

## 스레드

### 프로세스 : 실행 중인 프로그램
- 프로세스는 프로그램이 실행될 때마다 개별적으로 생성
- 하나의 프로세스는 프로그램을 수행함에 있어 필요한 데이터와 메모리 등의 할당 받은 자원, 그리고 하나 이상의 스레드로 구성됨

### 스레드 : 실제로 프로세스 안에서 자원을 이용해서 실제 작업을 수행하는 작업단위
- 모든 프로세스는 하나 이상의 스레드를 가지며 각각 독립적인 작업단위를 가짐
- 실제 일하는게 스레드이기 때문에 무조건 하나 이상의 스레드가 필요

> 메인 스레드
- 메인 스레드가 main()메소드를 실행하며 시작 main()의 첫 코드부터 아래로 순차적으로 실행되고, return을 만나면 종료
- 필요에 의해 작업 스레드들을 만들어서 병열 코드 실행 가능(멀티 스레드를 이용한 멀티 태스킹)

> 프로세스 종료
- 싱글 스레드의 경우 메인 스레드(main()메소드)가 종료되면 프로세스도 종료되지만, 멀티 스레드인 경우 실행 중인 스레드가 하나라도 있으면 프로세스가 종료되지 않음

> 멀티 프로세스 VS 멀티 스레드
- 멀티 프로세스 : 각각의 프로세스를 독립적으로 실행하는 것
- 멀티 스레드 : 하나의 프로세스 내에서 여러 스레드가 동시에 작업을 수행하는 것

> 싱글 스레드 : 메인스레드 하나만 가지고 작업을 처리 -> 한 작업씩 차례대로 처리해 나감

> 멀티 스레드 : 메인 스레드 외의 추가적인 스레드를 이용하여 별렬적으로 작업 처리
		하나의 스레드가 작업을 끝나면 추가 스레드가 일하는 것을 반복

> 멀티 스레드 장점 
- 자원을 보다 효율적으로 사용 가능
- 사용자의 대한 응답성 향상
- 애플리케이션의 응답성 향상
- 작업이 분리되어 코드가 간결해짐
- CPU 사용률 향상

> 멀티 스레드 단점
- 동기화(Synchonization)에 주의
	- 하나의 자원을 가지고 가서 스레드가 할당 된 자원을 처리해야 하는데 하나의 자원을 여러 스레드가 공유하고 있기 때문에 값이 변할 수 있기 때문에 이를 예방하고자 하나의 자원에 하나의 스레드만 접근할 수 있게 하는 것이 `동기화`
- 교착상태(dead-lock)가 발생하지 않도록 주의
	- 두 개 이상의 작업이 끝나길 기다리다가 아무것도 하지 못하는 것을 `교착 상태`라고 함

### 스레드 생성 
- Thead클래스 상속 : 객체 생성만으로 사용 가능
- Runnable 인터페이스 구현 : Thread 객체 생성 후 thread에 한 번 더 넣어줘야 함
> thread.start() 메소드가 Thread안에 있기 때문에 객체 생성 후 바로 사용 가능하나 Runnable에는 start()메소드가 없기 때문에 객체 생성 후 Thread 클래스 객체를 만들어서 레퍼런스 변수를 매개변수로 넣어서 start()변수 사용

### run()과 start()
- run() 호출 시 메모리의 Main Thread에 main()메소드 위로 run()메소드가 쌓임
- start() 호출 시 Main Thread에 main()에서 start()를 호출하고 start() 메소드가 새로운 스레드를 생성하고 그 위에 run()메소드가 쌓임

### 스레드 스케줄링
- 스레드 개수가 코어의 수보다 많을 경우 스레드를 어떤 순서로 동시성을 실행할 것인가를 결정하는 것으로 스케줄링에 의해 스레드들은 번갈아가며 run()메소드를 조금씩 실행

> 우선 순위 방식
- 우선 순위가 높은 스레드가 작업 시간을 더 많이 가지도록 하는 스케쥴링 방식
- 스레드에 1~10까지 우선 순위 번호 부여 가능 (번호가 높을 수록 우선 순위가 높음)
- 스레드 생성 시 우선 순위 기본값은 5

> 순환 할당 방식
- 시간 할당량을 정하여 하나의 스레드를 정해진 시간만큼 실행시키는 방식
- JVM에 의해 정해짐(코드로 제어 불가능)
