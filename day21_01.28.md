### 스레드 컨트롤
- 실행 중인 스레드의 상태를 제어하기 위한 것으로 효율적이고 정교한 스케줄링을 위한 스레드 상태를 제어하는 기능

스레드 생성 -> 실행대기 -> (일시정지) -> 실행 -> 스레드 소멸

- sleep() : 지정된 시간동안 스레드를 일시정지 시킴
- join() : 자신이 하던 작업을 잠시 멈추고 다른 스레드가 지정된 시간동안 실행되도록 함. 내가 지금 실행되고 있는 위치로 확인 가능
- wait() : 동기화된 블록 안에서 다른 스레드가 이 객체의 notify(), nptifyAll()을 호출하거나 지정된 시간이 지날 때 까지 현재의 스레드를 대기시킴

- sleep(), join(), wait(), I/O block를 만나면 일시정지에서 interrupt(),notify(), notifyAll()을 만나면 실행대기로 다시 가서 실행함 
- 실행 중 yield() 만나면 양보해서 실행대기로 돌아감

### 동기화
- 한 번에 한 개의 스레드만 프로세스 공유 자원(객체)에 접근할 수 있도록 락(Lock)을 걸어 다른 스레드가 진행 중인 작업에 간섭하지 못하도록 하는 것
- synchronized `메소드`나 `블록`에 사용해서 lock을 걸어줌 


## 네트워크 
- 여러 대의 컴퓨터를 통신 회선으로 연결한 것

### 서버와 클라이언트 : 네트워크로 연결된 컴퓨터간의 관계를 역할로 구분한 개념
- 서버 : 서비스를 제공하는 프로그램으로 클라이언트의 연결을 수락하고 요청 내용을 처리 후 응답을 보내는 컴퓨터
- 클라이언트 : 서비스를 제공 받는 프로그램으로 네트워크 데이터를 필요로 하는 모든 컴퓨터

### IP주소
- 네트워크 상에서 컴퓨터를 식별하는 번호

### 포트
- 같은 컴퓨터 내에서 프로그램을 식별하는 번호로 클라이언트는 서버 연결 요청 시 IP주소와 포트 번호를 알아야 함

### cmd 명령어
- path : 자바 설정 시 환경변수 path가 무엇으로 설정되어 있는지 보여주는 명령어
- ipconfig : 컴퓨터 TCP/IP를 보여주는 명령어
- cd(change directory) : 현재 내 디렉토리 위치/ 폴더를 이동함
- dir : 지금 디렉토리에 있는 파일과 하위 디렉토리 목록을 보여줌
- cls : 콘솔창 화면에 표시되어 있는 것들 지워줌

### InetAddress 클래스
- IP주소를 다루기 위해 자바에거 제공하는 클래스

### 소켓 프로그래밍
- 소켓을 이용한 통신 프로그래밍

#### 소켓
- 프로세스 간의 통신에 사용되는 `양쪽 끝 단`

#### TCP
- 데이터 전송 속도가 느리지만 정확하고 안정적으로 전달할 수 있는 연결 지향적 프로토콜. 신뢰성에 중점을 둠

#### UDP
- 데이터 전송 속도가 빠르지만 신뢰성 없는 데이터를 전송하는 비연결 지향적 프로토콜

### TCP 소켓 프로그래밍
- 서버가 먼저 실행되어 클라이언트의 요청이 올 때까지 기다려야하고 서버용 프로그램과 클라이언트용 프로그램을 따로 구현해야 함. 자바에서는 TCP 소켓 프로그래밍을 위해 java.net 패키지에서 ServerSoket과 Socket 클래스(클라이언트 쪽 Socket과 서버 쪽 Socket) 제공
- 서버의 Server Socket : 클라이언트 소켓에서 연결요청이 서로 연결하는 역할 / 응답대기 상태에서 클라이언트로 요청시 `accept()로 수락`을 하면 통신용의 Socket이 생김 
- 서버의 (Client) Socket : 클라이언트 소켓과 실제 데이터를 주고 받는 역할 / 클라이언트 소켓 객체 생성 

#### 서버용 TCP 소켓 프로그래밍 순서
1. 서버의 포트번호 정함
2. 서버용 소켓 객체(Server Socket) 생성
3. 클라이언트 쪽에서 접속 요청이 올 때까지 기다림
4. 접속 요청이 오면 요청 수락 후 해당 클라이언트에 대한 소켓 객체(Soket) 생성
5. 연결된 클라이언트와 입출력 스트림 생성
6. 보조 스트림을 통해 성능 개선
7. 스트림을 통해 읽고 쓰기
8. 통신 종료

#### 클라이언트용 TCP 소켓 프로그래밍 순서
1. 서버의 IP주소와 서버가 정한 포트번호를 매개변수로 하여 클라이언트용 소켓 객체 생성
2. 서버와의 입출력 스트림 오픈
3. 보조 스트림을 통해 성능 개선
4. 스트림을 통해 읽고 쓰기
5. 통신 종료
